UX 走查 Agent 架构总览

自动化 UX 体验走查系统：设计师提交表单，Claude AI 驱动无头浏览器访问目标网站， 从多个用户画像视角逐站评估，最终输出结构化 HTML 走查报告。

3 文件地图

server.js web/
Express HTTP 服务器。路由：POST /jobs、GET /api/jobs、GET /reports/:id。认证中间件验证 Supabase JWT。
worker.js web/
任务执行引擎。读取配置、启动 Playwright、遍历站点、调用 Claude generateText() 配合浏览器工具。
tools.js web/
定义 7 个 Playwright 工具供 Claude 使用：navigate、click、type、screenshot、evaluate、recordIssue、scroll。
checker.js web/
WalkthroughChecker 类。跟踪站点进度，分类错误（可重试/致命/轻微），决定是否中断走查。
db.js web/
Supabase CRUD 封装：createJob、getJob、updateJob、listJobs、incrementIssueCount。
config-writer.js web/
从 instances/alibaba-b2b/ 模板复制文件到 instances/{jobId}/，按选择筛选画像，生成 config.md。
models.js web/
Claude 模型注册表。将友好名称（claude-sonnet、claude-opus）映射到模型 ID。
supabase.js web/
初始化 Supabase 管理客户端（service role）和匿名客户端，读取环境变量。

实例模板

system_instructions.md
Agent 角色定义、评估规则、截图规范、严重度等级、输出格式。
personas.md
3 个默认画像：Lisa（新买家）、Carlos（经验买家）、Marcus（B2B 采购经理）。按任务筛选。
task_card_01_core.md
9 站点路线图。每站包含：目标页面、操作步骤、多画像评估维度、截图要求。
issue_schema.md
结构化问题格式：id、严重度（P0-P3）、分类（Quick Win / 结构性 / 战略性）、影响、建议。

4 Worker 执行循环

每个任务在 worker.js 中按以下流程执行：

// 简化的执行流程

for (const station of stations) {
  // 1. 创建新的浏览器上下文（1440x900 视口）
  const context = await browser.newContext({ viewport: { width: 1440, height: 900 } });

  // 2. 构建站点专属 Prompt
  const prompt = `执行站点 ${station.id}: ${station.name}\n${station.content}`;

  // 3. 调用 Claude，附带 Playwright 工具
  await generateText({
    model: 'claude-sonnet-4-20250514',
    system: [systemInstructions, issueSchema],
    prompt: prompt,
    tools: [navigate, click, type, screenshot, evaluate, recordIssue, scroll],
    maxSteps: 50,
    maxTokens: 4096,
  });

  // 4. 可重试错误最多重试 3 次（超时、限流）
  // 5. 轻微错误跳过当前站（元素未找到）
  // 6. 累计 5+ 致命错误则终止整个走查

  // 7. 更新数据库进度（触发 Realtime 推送到前端）
  await updateJob(jobId, { progress: checker.getProgress() });
}

5 Claude 的浏览器工具

工具用途关键参数
navigate 跳转到 URL url, waitUntil (load/domcontentloaded/networkidle)
click 点击元素 selector（CSS 选择器）或 text（可见文本匹配）
type 在输入框中输入 selector, text, pressEnter
screenshot 截取页面/元素 filename, selector（可选）, fullPage
evaluate 在页面中执行 JS script（任意 JavaScript）
recordIssue 保存结构化问题 id, severity, title, description, personas, screenshot
scroll 滚动视口 direction (up/down), pixels

8 错误分类

错误类型示例处理方式
可重试 超时、限流、网络错误 最多重试 3 次，递增退避
致命 认证失败、浏览器崩溃、目标关闭 跳过当前站，累计致命计数器。达到 5 次则终止走查。
轻微 元素未找到、Cookie 弹窗 跳过当前站，继续走查

最终状态判定

  • done 所有站点均已完成
  • partial 部分站点完成，部分有致命错误
  • failed 无站点完成，或被致命错误阈值中断
