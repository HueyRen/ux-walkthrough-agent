# B2B 电商体验走查评估规则库

## 使用说明

本规则库按用户旅程阶段组织，每条规则带有适用标签。
走查系统会根据当前场景的画像、品类、行业自动匹配适用规则，减少不相关规则的干扰。

**匹配逻辑:**
- `persona:*` 表示适用所有画像；`persona:small_retail` 表示仅适用该画像
- 多个标签之间为 AND 关系（同时满足才匹配）；同类型多个标签之间为 OR 关系
- `priority:critical` 规则在所有走查中必须评估，不可跳过

**严重度评级定义:**
- **好 (Good):** 完全满足规则标准，用户无明显阻力
- **可接受 (Acceptable):** 基本满足，存在轻微瑕疵但不影响任务完成（记为 P3）
- **有问题 (Issue):** 明显不满足，用户需要额外认知成本或操作步骤（记为 P1/P2）
- **严重问题 (Critical):** 阻断用户任务，或造成信任损失、决策错误（记为 P0）

---

## 标签说明

| 标签类型 | 可选值 | 说明 |
|----------|--------|------|
| `persona:*` | `small_retail`, `medium_brand`, `large_brand`, `service_provider`, `manufacturer`, `*`（全部） | 适用买家画像 |
| `category:*` | `standard`, `light_custom`, `heavy_custom`, `*`（全部） | 适用品类类型 |
| `industry:*` | `parameter_type`, `visual_type`, `*`（全部） | 适用行业类型（参数型 vs 视觉型） |
| `experience:*` | `veteran`, `novice`, `experienced_new` | 适用经验水平（老手/新手/有经验的新平台用户） |
| `priority:*` | `critical`, `high`, `medium`, `low` | 规则优先级 |

## 找 (Find) 阶段规则

> 目标：用户能否从首页/搜索入口快速定位到目标商品类目和潜在供应商

### F-01: 首页导购信息架构

```
[tags: persona:*, category:*, industry:*, priority:critical]
```

**规则目标:** 首页的品类导航和导购入口是否帮助不同画像用户快速定向，减少无效浏览。

**检查方法:**
1. 打开首页，不做任何操作，截图保存初始状态
2. 观察一级导航/顶部 Category 菜单的展开方式（hover? click? 默认展开?）
3. 展开品类导航，记录一级品类数量、二级品类是否可见
4. 检查首页 Banner/推荐模块：是否有行业/场景导购入口（如「For Retailers」「Electronics」专区）
5. 以新用户视角：5 秒内能否判断该平台是否覆盖自己要找的品类？

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 一级品类覆盖全面，二级/三级在 2 次交互内可达；首页有面向不同买家角色的明确入口；新用户 5 秒内可定向 |
| 可接受 | 品类结构清晰但需要 3–4 次点击才能到目标品类；首页信息略多但不干扰主流程 |
| 有问题 | 品类导航层级混乱，或关键品类藏得太深；首页以促销内容为主，导购功能弱化 |
| 严重问题 | 找不到目标品类入口，或品类名称使用平台内部术语导致理解困难 |

**截图要求:**
- 必截：首页初始状态（无任何交互）
- 必截：品类导航展开后的完整层级
- 如有问题：截图标注找不到的入口位置

**画像特别关注:**
- `small_retail` (novice): 可能不知道行业术语（如「OEM」「私模」），检查品类名称是否使用通俗语言
- `large_brand` (veteran): 检查是否有面向企业采购的专属入口（如 Alibaba.com 的「Enterprise Sourcing」）
- `manufacturer` (veteran): 检查工业品/原材料的专区是否显眼，或被消费品导购淹没
- `service_provider` (experienced_new): 检查是否有代理采购 / 多账号管理的入口提示

### F-02: 搜索结果相关性与排序逻辑

```
[tags: persona:*, category:*, industry:*, priority:critical]
```

**规则目标:** 搜索结果是否准确反映用户查询意图，排序逻辑是否对买家透明且合理。

**检查方法:**
1. 使用代表性关键词搜索（为每个走查画像准备 2–3 个典型搜索词）
2. 截图第一屏（不滚动）的搜索结果
3. 逐条检查前 10 条结果：标题是否匹配、品类是否准确、是否有明显无关结果
4. 检查排序说明：「Best Match」/「Most Popular」/「Newest」等是否有说明其含义
5. 测试同一关键词使用行业俗语 vs 标准术语的结果差异（如「t-shirt」vs「polo shirt」vs「POLO衫」）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 前 10 条结果相关度 ≥ 9/10；排序方式有清晰说明；拼写错误有智能纠正 |
| 可接受 | 前 10 条相关度 7–8/10；有 1–2 条明显广告但标注清晰；排序逻辑虽不完全透明但结果合理 |
| 有问题 | 前 10 条中有 3+ 条明显不相关；广告/非相关结果未标注；搜索结果与用户预期品类偏差大 |
| 严重问题 | 返回空结果但实际平台有该品类商品；或结果完全不相关导致用户认为平台不覆盖该品类 |

**截图要求:**
- 必截：搜索结果第一屏（含搜索框、搜索词、前 6–8 条结果）
- 如有问题：标注不相关结果的位置
- 如测试多个搜索词：每个词各截一张

**画像特别关注:**
- `small_retail` (novice): 测试用通俗名称搜索的结果（他们不会用行业术语）
- `manufacturer` (veteran): 测试使用材料规格、型号编码搜索的结果（如「304 stainless steel coil 0.5mm」）
- `service_provider` (experienced_new): 测试跨品类组合搜索（如「furniture accessories」）
- `visual_type` 行业: 测试用描述性词语搜索的结果（如「boho style dress」）

### F-03: 筛选器对品类参数的覆盖度

```
[tags: persona:medium_brand, persona:large_brand, persona:manufacturer, category:standard, category:light_custom, industry:parameter_type, priority:high]
```

**规则目标:** 搜索结果页的筛选器是否覆盖该品类的核心采购决策参数，支持精准缩圈。

**检查方法:**
1. 在目标品类下执行搜索，进入结果页
2. 展开所有筛选器面板，截图完整筛选项列表
3. 对照该品类的核心采购参数（走查前预设），逐一检查是否有对应筛选项
4. 测试组合筛选（勾选 2–3 个条件），检查结果数量是否合理
5. 检查筛选项的值范围是否合理（如 MOQ 范围、价格区间的粒度）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 覆盖该品类 80%+ 核心参数；筛选项有分组（材料/规格/认证等）；组合筛选结果准确 |
| 可接受 | 覆盖 60–79% 核心参数；部分参数需通过搜索词补充；筛选结果偶有不准确 |
| 有问题 | 筛选项以通用维度为主（价格/地区/认证），缺乏品类专属参数；用户必须逐条看详情才能判断 |
| 严重问题 | 筛选器完全缺失核心采购参数，导致用户无法缩减结果范围 |

**截图要求:**
- 必截：完整筛选器面板（若需多屏，截图全部）
- 如有问题：标注缺失的关键筛选项（可在截图外标注文字说明）

**画像特别关注:**
- `parameter_type` 行业（五金/电子/机械）: 重点检查规格参数筛选（材质、厚度、电压、认证标准）
- `visual_type` 行业（服装/家居/工艺品）: 检查颜色/风格/季节/场合等视觉筛选项
- `manufacturer`: 检查材料等级、供货周期、起订量范围是否可筛选
- `large_brand`: 检查是否可筛选「Trade Assurance」「Verified Supplier」等信任认证

### F-04: 平台专业术语理解门槛

```
[tags: persona:small_retail, persona:experienced_new, experience:novice, category:*, priority:high]
```

**规则目标:** 平台特有术语（Trade Assurance、MOQ、RFQ、Gold Supplier 等）是否有充分说明，不造成新用户的认知障碍。

**检查方法:**
1. 在搜索结果页和商品详情页，列举所有出现的平台术语
2. 对每个术语，检查：是否有 tooltip / 链接 / 说明文字？
3. 点击「Trade Assurance」图标或文字，检查说明页面的清晰度
4. 检查「RFQ」（Request for Quotation）入口：是否说明了 RFQ 的用途和流程？
5. 检查「Gold Supplier」/「Verified Supplier」徽章：悬停或点击是否解释含义和认证标准？

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 所有专业术语有内联说明（tooltip 或括号注释）；点击可查看详细解释；说明语言对新手友好 |
| 可接受 | 主要术语有说明，但需要点击跳转才能看到；说明内容专业但能理解 |
| 有问题 | 多数术语无说明，或说明需要跳转多层；新手必须依赖外部搜索才能理解 |
| 严重问题 | 关键流程（如 Trade Assurance 如何保护买家）没有说明，或说明与实际功能不符 |

**截图要求:**
- 必截：出现较多术语的页面区域（如结果页的供应商信息区）
- 如有说明：截图 tooltip 或说明页面
- 如无说明/说明不清：截图并标注具体术语位置

**画像特别关注:**
- `small_retail` (novice): 重点关注 MOQ、Trade Assurance、RFQ 这三个对其决策影响最大的术语
- `experienced_new`（有 B2B 经验但首次用该平台）: 关注平台独有的术语（如 Alibaba 的「Smart Matching」）
- `service_provider`: 关注代理采购相关术语（如 Sourcing Agent、Consolidation）

### F-05: 推荐逻辑相关性

```
[tags: persona:small_retail, persona:medium_brand, category:*, priority:medium]
```

**规则目标:** 首页/搜索结果中的推荐商品是否与用户当前情境相关，而非纯商业化曝光。

**检查方法:**
1. 在首页和搜索结果页，识别所有「推荐」/「You May Also Like」/「Sponsored」等推荐位
2. 对每个推荐模块：检查推荐商品与当前搜索意图的相关度
3. 检查「Sponsored」标注是否清晰（广告披露）
4. 在搜索某品类后，检查页面右侧或底部的「Related Categories」推荐是否准确
5. 若有个性化推荐（登录状态），检查是否基于浏览历史（合理的上下文关联）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 推荐内容与当前情境高度相关（相同品类或互补品类）；广告标注清晰不混淆 |
| 可接受 | 推荐内容大体相关但有少量偏差；广告标注存在但不够突出 |
| 有问题 | 推荐内容与当前情境无关（如搜索电子产品推荐服装）；推荐位占用过多页面空间 |
| 严重问题 | 推荐内容干扰主流程（如遮挡搜索结果）；或推荐明显误导用户决策 |

**截图要求:**
- 按需截图：当推荐内容明显不相关时截图记录
- 如发现广告未标注：截图标注具体位置

**画像特别关注:**
- `small_retail`: 检查新用户首次访问时的推荐是否有「引导」功能（帮助发现平台能力）
- `medium_brand`: 检查是否有基于其采购类型的精准推荐（如上次搜了包装材料，推荐配套印刷供应商）

### F-06: AI 搜索 / AI Mode 的可发现性与质量

```
[tags: persona:*, category:*, priority:high]
```

**规则目标:** 平台的 AI 搜索功能（如 AI Smart Matching、AI-powered RFQ）是否易于发现，且搜索质量能真正帮助用户缩短找货路径。

**检查方法:**
1. 在首页和搜索框区域，识别 AI 相关功能入口（AI 搜索、智能询盘、AI 选品等）
2. 检查入口的视觉显著性：是否需要主动寻找？是否有引导提示？
3. 激活 AI 搜索功能，输入自然语言查询（如「I need eco-friendly packaging for cosmetics under $500 MOQ」）
4. 评估 AI 返回结果的质量：是否理解了多条件查询？是否提供了有针对性的结果？
5. 检查 AI 结果中是否有「原因说明」（为什么推荐这些结果/供应商）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | AI 搜索入口在首屏显眼位置；支持自然语言多条件输入；结果有质量说明；比普通搜索节省 ≥ 2 步操作 |
| 可接受 | AI 功能存在但入口不够显眼；能处理简单自然语言查询；结果基本准确但无说明 |
| 有问题 | AI 功能藏在二级页面；或仅支持关键词搜索（与普通搜索无差异）；结果质量不稳定 |
| 严重问题 | AI 搜索功能不可用或报错；或 AI 结果完全不相关，误导用户 |

**截图要求:**
- 必截：AI 搜索入口的可见状态（首屏）
- 必截：输入自然语言查询后的结果页
- 如有问题：截图 AI 理解错误或结果不相关的具体案例

**画像特别关注:**
- `small_retail` (novice): 检查 AI 功能是否有新手引导（示例查询、提示词提示）
- `large_brand` (veteran): 检查 AI 是否支持复杂条件（多品类、认证要求、供货量要求）
- `service_provider`: 检查是否支持批量 AI 询盘（多 SKU 同时处理）

### F-07: 图搜功能可用性

```
[tags: persona:small_retail, persona:medium_brand, industry:visual_type, category:*, priority:medium]
```

**规则目标:** 以图搜图功能是否易于发现、操作流畅，搜索结果与参考图的相似度满足采购需求。

**检查方法:**
1. 在搜索框区域，寻找图搜入口（相机图标 / 「Search by Image」）
2. 检查入口的视觉可见性（是否需要悬停才能发现？）
3. 上传一张代表性商品图片（准备一张中等复杂度的产品图），截图上传过程
4. 评估搜索结果：前 10 条中有多少条与参考图在风格/品类上高度相似？
5. 检查是否支持「框选图片局部搜索」或「相似商品推荐」等进阶功能

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 图搜入口在搜索框显眼位置；上传图片 < 3 步；前 10 条结果相似度 ≥ 7/10 |
| 可接受 | 图搜入口需要寻找但能找到；上传步骤 3–5 步；前 10 条相似度 5–6/10 |
| 有问题 | 图搜入口很难发现（需超过 2 次探索）；或上传后长时间无响应；结果相似度低 |
| 严重问题 | 图搜功能不存在或无法使用；或返回完全无关结果 |

**截图要求:**
- 必截：图搜入口的初始状态（图标位置）
- 必截：上传图片后的搜索结果第一屏
- 如有问题：截图具体卡点步骤

**画像特别关注:**
- `visual_type` 行业用户（服装/家居）: 这是高频需求，图搜失败对其伤害最大
- `small_retail` (novice): 检查是否有使用提示（「Upload a product photo to find similar items」）
- `service_provider`: 检查是否支持批量图搜（多图同时搜索）

### F-08: 分类浏览 vs 搜索路径的平衡

```
[tags: persona:small_retail, persona:medium_brand, experience:novice, category:*, priority:medium]
```

**规则目标:** 平台是否同时支持「我知道要什么」（搜索驱动）和「我想探索发现」（浏览驱动）两种用户心智，且两条路径都顺畅。

**检查方法:**
1. 完全不使用搜索功能，从首页通过分类浏览进入目标品类，记录点击次数和路径
2. 检查分类页面的商品展示密度和信息丰富度（浏览模式下，缩略图+关键参数是否足够决策）
3. 检查「探索型」用户的导购支持：是否有「Trending」/「Popular」/「New Arrivals」等浏览入口？
4. 比较搜索路径 vs 浏览路径达到同一目标品类的步骤数量差异
5. 检查两条路径的结果集是否一致（搜索结果 vs 分类浏览结果，是否同一批商品）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 浏览路径 ≤ 3 步到达目标品类；浏览模式信息密度足够决策；两条路径结果一致且互补 |
| 可接受 | 浏览路径 4–5 步；分类页信息基本够用；偶有路径不一致但可接受 |
| 有问题 | 浏览路径超过 5 步或迷路；分类页信息不足以初步筛选；两条路径结果差异大 |
| 严重问题 | 浏览路径存在死路（无法返回或分类错误）；或平台严重依赖搜索，浏览几乎不可用 |

**截图要求:**
- 按需截图：浏览路径中的关键卡点（找不到下一步时）
- 如发现两路径结果不一致：分别截图对比

**画像特别关注:**
- `small_retail` (novice): 新手更依赖浏览探索，检查分类入口是否足够直观
- `medium_brand` (veteran): 他们倾向于搜索，检查搜索结果是否比浏览结果更丰富

## 挑 (Select) 阶段规则

> 目标：用户在目标商品的详情页能否获得足够的信息做出采购决策，包括商品信息、供应商资质和比较便利性

---

### S-01: 商品详情页信息完整度

```
[tags: persona:*, category:*, industry:*, priority:critical]
```

**规则目标:** 商品详情页是否提供采购决策所需的所有关键信息，无需用户主动询问即可完成初步决策。

**检查方法:**
1. 打开目标商品详情页，截图完整页面（分段截图）
2. 对照该品类的必要信息清单，逐项检查是否存在且清晰：
   - 商品标题（是否含规格关键词）
   - 价格 / 价格区间（含 MOQ 说明）
   - 主图 + 多角度图片
   - 商品规格参数表（材质/尺寸/重量/认证等）
   - 库存 / 供货能力说明
   - 交货期（Lead Time）
   - 包装信息（装箱量/包装方式）
   - 供应商基本信息（工厂/贸易商/位置）
3. 检查信息的「层级」：是否有「核心信息优先」的信息架构（重要信息在首屏，细节在后）？

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 80%+ 必要信息完整且在合理位置；首屏包含价格、规格摘要、主图；信息层级清晰 |
| 可接受 | 60–79% 信息完整；部分信息需要滚动较远才能找到；信息层级基本合理 |
| 有问题 | 多项关键信息缺失（如无交货期说明/无包装信息）；或信息散乱无层级 |
| 严重问题 | 核心采购信息（价格/规格/供货能力）缺失或严重不准确 |

**截图要求:**
- 必截：详情页首屏（不滚动）
- 必截：规格参数区域
- 如有缺失：截图并标注缺少哪些信息

**画像特别关注:**
- `small_retail` (novice): 检查是否有「How to Order」「What's Included」等新手友好说明
- `large_brand` (veteran): 检查是否有工厂产能数据、认证原件等企业采购所需信息
- `manufacturer`: 检查原材料/工业品是否有材料规格书（Material Spec Sheet）下载
- `parameter_type` 行业: 重点检查参数表的完整性和准确性

### S-02: 供应商可信度信号的可见性

```
[tags: persona:*, category:*, priority:critical]
```

**规则目标:** 供应商的可信度信号（认证、年限、评价、Trade Assurance 等）是否在详情页显眼展示，帮助买家快速建立或排除信任。

**检查方法:**
1. 在详情页，识别所有与供应商可信度相关的信号元素
2. 逐一检查可见性：首屏可见 / 需滚动 / 找不到
3. 检查每类信号的可操作性：点击「Verified Supplier」是否跳转到验证详情？
4. 检查负面信号：差评、低回复率、近期投诉是否也有展示（或是否被刻意隐藏）？
5. 检查供应商主页入口：从详情页到供应商主页的路径是否清晰？

**可信度信号清单（核查用）:**
- Trade Assurance 保障金额
- Verified Supplier / Gold Supplier 徽章
- 经营年限（Years in Business）
- 响应率（Response Rate）和响应时长
- 买家评价（Rating + Review 数量）
- 第三方认证（ISO / CE / FDA 等）
- 工厂视频/审厂报告

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 5+ 信号在首屏或 1 屏滚动内可见；信号可点击查看详情；负面信号不被隐藏 |
| 可接受 | 3–4 个主要信号可见；部分信号需点击展开；负面信号有但展示不突出 |
| 有问题 | 仅 1–2 个信号可见且需滚动找；或信号堆砌但无法点击了解详情 |
| 严重问题 | 核心信任信号（评价/认证）缺失；或存在明显虚假信号（零评价但显示「Top Supplier」） |

**截图要求:**
- 必截：供应商信息区域（含所有可见的信任信号）
- 如有问题：截图并标注信号位置和可见性问题

**画像特别关注:**
- `small_retail` (novice): 对 Trade Assurance 最敏感，检查保障说明是否清晰（能保多少钱、什么情况下赔）
- `large_brand`: 对第三方审厂报告和 ISO 认证要求更高，检查是否可在详情页获取
- `service_provider`: 关注响应率和响应时长——代理采购需要快速回应的供应商

[tags: persona:*, industry:visual_type, category:standard, category:light_custom, priority:high]

**检查方法:**

**截图要求:**

- 如有问题：截图模糊图片或信息遮挡情况

**画像特别关注:**
- `visual_type` 行业：图片质量对其决策权重最高，对该画像此规则升级为 `priority:critical`
- `small_retail` (novice): 可能不会主动寻找视频，检查视频是否自动播放或有明显入口
- `manufacturer`: 工业品图片更关注尺寸图和技术参数图，而非美观度

### S-04: 价格透明度

```
[tags: persona:*, category:*, priority:critical]
```

**规则目标:** 商品价格展示是否清晰、诚实，能帮助用户快速判断是否在预算范围内；定制品是否有明确的询价引导。

**检查方法:**
1. 在搜索结果页，检查价格显示方式：显示单价还是批量价？含税还是不含税？
2. 进入详情页，检查价格区间：是否有阶梯价（Tiered Pricing）说明（如 50pcs: $X.XX, 100pcs: $X.XX）
3. 检查价格是否包含物流费（FOB/EXW/DDP 说明是否清晰）
4. 对于定制品：是否有「Contact for Price」或询价表单引导，且说明为什么需要询价？
5. 检查「最低起订量（MOQ）」与价格的关联展示是否清晰

**评级标准（按品类区分）:**

**标准品 (standard):**
| 评级 | 标准 |
|------|------|
| 好 | 明确显示单价/批量价；阶梯价清晰可读；贸易条款（FOB等）标注；MOQ 与价格联动展示 |
| 可接受 | 价格可见但需要计算换算；贸易条款有标注但不显眼 |
| 有问题 | 价格区间过大无实际参考价值（如「$0.10 – $50.00」）；或需要登录才能看价格 |
| 严重问题 | 不显示价格或显示明显虚假低价（诱导入详情再改价） |

**定制品 (light_custom / heavy_custom):**
| 评级 | 标准 |
|------|------|
| 好 | 提供参考价格区间 + 说明影响价格的因素；询价入口显眼且说明询价后的流程和时效 |
| 可接受 | 提供大致价格范围；询价入口可找到 |
| 有问题 | 完全不显示价格参考，也不说明为什么；询价入口隐藏 |
| 严重问题 | 用标准品价格误导定制品决策；或询价流程不存在 |

**截图要求:**
- 必截：价格展示区域（含阶梯价/贸易条款/MOQ）
- 如有问题：截图价格歧义或缺失位置

**画像特别关注:**
- `small_retail`: 对 MOQ 最敏感，检查 MOQ 是否在首屏显眼位置
- `large_brand`: 关注是否有「批量折扣谈判」入口（采购量大但想议价）
- `service_provider`: 经常代多个客户采购不同 MOQ，检查阶梯价的粒度是否够细

### S-05: 商品比较的便利性

```
[tags: persona:medium_brand, persona:large_brand, persona:service_provider, category:standard, priority:medium]
```

**规则目标:** 用户是否可以便捷地对比多个候选供应商/商品，而无需在多个标签页之间切换记忆。

**检查方法:**
1. 在搜索结果页，寻找「Compare」或「Add to Compare」功能
2. 选择 3–4 个商品，激活比较功能，截图比较页面
3. 检查比较维度的覆盖度（是否包含价格/规格/认证/评价等核心维度）
4. 检查比较页面的可读性（信息是否对齐，差异是否突出）
5. 若无内置比较功能，检查是否有「Wishlist」/「Inquiry Cart」等批量操作替代方案

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 内置比较功能，支持同时比较 ≥ 3 个商品；比较维度覆盖核心采购参数；差异项自动高亮 |
| 可接受 | 有比较功能但支持商品数量有限（仅 2 个）；或维度覆盖不全但主要参数有 |
| 有问题 | 无内置比较功能，但有 Wishlist 可批量联系；比较功能存在但数据不准确 |
| 严重问题 | 完全无比较或收藏功能，用户只能依赖浏览器多标签页 |

**截图要求:**
- 按需截图：激活比较功能后的比较页面
- 如无比较功能：截图 Wishlist 或替代方案

**画像特别关注:**
- `large_brand` (veteran): 采购决策需要更多比较维度，检查是否可自定义比较项
- `service_provider`: 代理采购需要快速比较多个方案，对比较功能依赖度高

### S-06: 定制选项的清晰度

```
[tags: persona:medium_brand, persona:large_brand, category:light_custom, category:heavy_custom, priority:high]
```

**规则目标:** 定制化选项（颜色/尺寸/Logo/材料选择等）是否展示清晰，用户能理解定制的范围、限制和对价格的影响。

**检查方法:**
1. 在支持定制的商品详情页，找到所有定制选项（颜色选择/尺寸规格/Logo 定制等）
2. 检查每个定制选项：是否有视觉预览（选颜色时有色板或预览图）？
3. 检查定制选项对价格的影响：更改选项时价格是否实时更新？
4. 检查定制的「边界说明」：有哪些选项是固定不可改的？有哪些需要额外费用？
5. 对于重定制（OEM/ODM）：检查是否有「定制流程说明」（设计稿 → 打样 → 确认 → 批量生产）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 所有定制选项有视觉预览；价格随选项实时更新；定制边界清晰说明；OEM 流程可见 |
| 可接受 | 主要定制选项有说明；价格变化有提示但不实时；定制限制基本说明清楚 |
| 有问题 | 定制选项仅文字描述，无预览；不知道哪些选项影响价格；定制流程不清晰 |
| 严重问题 | 标注「支持定制」但实际无任何定制引导；或定制选项与实际产品严重不符 |

**截图要求:**
- 必截：定制选项区域（含所有可选项）
- 如有实时价格更新：截图选项改变前后的价格对比
- 如有问题：截图定制流程缺失或选项模糊的位置

**画像特别关注:**
- `light_custom` 品类（如印刷包装、私标产品）: 重点检查 Logo/颜色定制的操作流程
- `heavy_custom` 品类（如模具制造、定制零件）: 重点检查 OEM 流程引导和设计文件上传入口
- `medium_brand`: 通常需要轻定制（改颜色/加标），检查操作是否顺畅

### S-07: MOQ 与价格阶梯展示

```
[tags: persona:small_retail, persona:medium_brand, category:*, priority:high]
```

**规则目标:** 最低起订量（MOQ）和批量价格阶梯是否展示清晰，帮助买家快速判断自身采购规模是否匹配。

**检查方法:**
1. 在搜索结果页，检查 MOQ 是否在卡片上直接显示（不用点击进详情）
2. 进入详情页，找到 MOQ 信息，检查是否清晰（数量 + 单位）
3. 检查价格阶梯（Tiered Pricing）：
   - 阶梯是否覆盖该品类的常见采购量范围？
   - 阶梯价格是否实时联动（输入数量后自动显示单价）？
4. 检查 MOQ 的灵活性说明：是否支持「混款/混色」达到 MOQ？是否可商议 MOQ？
5. 检查 MOQ 与库存的关系：「现货」商品的 MOQ 是否与「定制」商品有明显区分？

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | MOQ 在搜索结果卡片和详情页首屏均可见；价格阶梯 ≥ 3 档且覆盖常见采购量；支持数量输入联动价格 |
| 可接受 | MOQ 仅在详情页可见；价格阶梯 2 档；无实时联动但阶梯表格清晰 |
| 有问题 | MOQ 需滚动才能找到；价格阶梯仅 1 档（无批量优惠说明）；单位模糊（「1 piece」还是「1 box」?) |
| 严重问题 | MOQ 信息缺失；或显示 MOQ = 1 但实际联系后要求大量最低订购 |

**截图要求:**
- 必截：详情页的 MOQ 和价格阶梯区域
- 如有问题：截图 MOQ 信息模糊或缺失的位置

**画像特别关注:**
- `small_retail`: 对高 MOQ 最敏感，检查是否有「小批量」/「Ready to Ship」筛选，帮助其找到合适的供应商
- `manufacturer`: 工业原材料采购量通常大，检查大批量价格阶梯（如 1 吨/10 吨/100 吨）是否可见

### S-08: Ready to Ship vs 定制商品的区分清晰度

```
[tags: persona:small_retail, persona:medium_brand, category:standard, category:light_custom, priority:high]
```

**规则目标:** 用户能否快速区分「现货即发」和「需要定制/生产」的商品，避免因错误预期导致的交期问题。

**检查方法:**
1. 在搜索结果页，检查是否有「Ready to Ship」标签，以及标签的显著性
2. 进入含「Ready to Ship」标签的商品详情，检查现货数量、发货时效是否明确说明
3. 同时打开普通定制商品详情，对比两者的首屏视觉差异（能否一眼区分？）
4. 检查「Ready to Ship」商品的交付流程说明（下单 → 发货 → 物流 → 到货的时间预期）
5. 检查平台是否有专门的「Ready to Ship」频道/专区入口（帮助急需补货的买家快速找到现货）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 搜索结果页有明显「Ready to Ship」徽章；有专属筛选/频道；详情页清晰说明库存量和发货时效 |
| 可接受 | 搜索结果有标签但不突出；详情页有说明但需滚动找到；无专属频道但可筛选 |
| 有问题 | 仅在详情页才能得知是否现货；无筛选功能；现货和定制商品视觉上无明显区分 |
| 严重问题 | 「Ready to Ship」标签与实际库存不符（有货标签但实际需要生产）；或现货/定制完全混淆 |

**截图要求:**
- 必截：含「Ready to Ship」标签的搜索结果对比（有标签 vs 无标签商品并排）
- 必截：Ready to Ship 详情页的发货时效区域
- 如有问题：截图现货/定制混淆的具体案例

**画像特别关注:**
- `small_retail`: 高度依赖 Ready to Ship（通常无法等待生产周期）；检查此画像是否能快速找到现货商品
- `medium_brand`: 部分订单急需补货，检查 Ready to Ship 商品的选择丰富度是否满足常见补货需求

### I-01: 询盘表单字段合理性

```
[tags: persona:*, category:*, priority:critical]
```

**规则目标:** 询盘/联系供应商的表单字段是否合理——既能收集供应商需要的信息，又不给买家造成过重的填写负担。

**检查方法:**
1. 点击「Contact Supplier」/「Send Inquiry」按钮，截图初始表单状态
2. 列举所有必填字段（Required）和选填字段（Optional）
3. 评估字段合理性：
   - 对于标准品：买家需要填的必要信息（数量/规格/目的地）是否都有？是否有冗余字段？
   - 对于定制品：是否有文件上传入口（设计稿/规格文档）？
4. 检查表单是否有智能预填（如自动填入商品名称、已知的买家信息）
5. 提交表单后，检查确认提示（发送成功/预计回复时间/后续步骤说明）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 必填字段 ≤ 5 个，且均与采购直接相关；有文件上传；智能预填商品信息；提交后有清晰确认和后续引导 |
| 可接受 | 必填字段 6–8 个，大部分与采购相关；无文件上传但有留言区；提交后有确认消息 |
| 有问题 | 必填字段 > 8 个（包含不必要的公司信息）；无文件上传；提交后无任何后续说明 |
| 严重问题 | 表单无法提交（技术错误）；或必填字段设计不合理（如强制要求公司注册号阻止个人采购） |

**截图要求:**
- 必截：完整询盘表单（含所有字段）
- 必截：提交成功后的确认页面/提示
- 如有问题：截图具体不合理字段或错误提示

**画像特别关注:**
- `small_retail` (novice): 字段越少越好，检查是否有过度的企业信息要求（阻止个体户询盘）
- `large_brand` (veteran): 需要详细填写规格要求，检查是否有「项目描述」大文本框和附件上传
- `manufacturer`: 工业品询盘通常需要上传技术图纸，检查是否支持 PDF/CAD 文件格式

### I-02: 沟通入口的可发现性

```
[tags: persona:*, category:*, priority:critical]
```

**规则目标:** 「联系供应商」的入口是否在详情页显眼且多路径可达，避免用户有意愿沟通却找不到入口。

**检查方法:**
1. 在商品详情页，不做任何操作，5 秒内能否发现「Contact Supplier」按钮？
2. 检查沟通入口的位置：首屏可见？需要滚动？在浮动 CTA 栏？
3. 检查多种沟通入口是否共存：即时聊天（Chat Now）/ 询盘表单（Send Inquiry）/ 电话 / WhatsApp？
4. 检查在滚动页面后，沟通入口是否始终保持可见（Sticky CTA）
5. 从供应商主页进入，检查「Contact」入口是否同样清晰

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 首屏可见且视觉突出（颜色对比强）；随滚动保持 sticky；提供 2+ 种沟通方式；入口文字清晰表达动作 |
| 可接受 | 首屏可见但不突出；无 sticky 但底部也有入口；至少 1 种主要沟通方式清晰 |
| 有问题 | 需滚动才能找到沟通入口；入口标签模糊（如仅显示供应商名称而无 CTA 文字） |
| 严重问题 | 沟通入口藏在三级导航或需要登录后才出现（在未登录状态下对新用户隐藏） |

**截图要求:**
- 必截：详情页首屏中的沟通入口区域（展示其可见性）
- 如有 sticky CTA：截图滚动后的状态
- 如有问题：截图找不到入口或入口不显眼的状态

**画像特别关注:**
- `small_retail` (novice): 检查是否有「How to contact supplier」的引导说明（新手可能不知道如何开始沟通）
- `service_provider`: 检查是否可以批量发送询盘给多个供应商（一次操作 vs 逐个操作）
- `large_brand`: 检查是否有「Corporate Account」专属沟通入口（如 Key Account Manager 联系方式）

### I-03: 沟通工具的能力（文件/图片传输支持）

```
[tags: persona:medium_brand, persona:large_brand, persona:service_provider, persona:manufacturer, category:light_custom, category:heavy_custom, priority:high]
```

**规则目标:** 平台内置的沟通工具（Chat）是否支持 B2B 采购场景所需的信息交换，包括文件、图片、规格文档的发送。

**检查方法:**
1. 进入与供应商的聊天界面，截图工具栏和输入区域
2. 测试文件发送：尝试发送一个 PDF 文件，检查是否支持、上传速度、预览效果
3. 测试图片发送：尝试发送产品参考图，检查是否支持多张同时发送
4. 检查是否支持语音/视频通话（对于复杂定制采购很重要）
5. 检查消息历史：聊天记录是否持久保存？是否可搜索历史消息？

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 支持文件（PDF/Excel/CAD）+ 图片发送；文件大小限制 ≥ 20MB；聊天记录持久存储且可搜索；支持语音/视频 |
| 可接受 | 支持图片发送，文件支持有限（仅 PDF）；历史记录保存但不可搜索；无视频通话但有联系方式分享 |
| 有问题 | 仅支持文字和图片，无文件传输；历史记录保存时间短（< 30 天）；沟通工具功能基础 |
| 严重问题 | 沟通工具不支持文件传输，买家需通过邮件补充（断链）；或聊天记录不保存 |

**截图要求:**
- 必截：聊天界面工具栏（展示支持的操作）
- 如有文件发送测试：截图发送过程和结果
- 如有问题：截图功能缺失或发送失败的情况

**画像特别关注:**
- `heavy_custom` 品类：经常需要发送技术图纸（DWG/STEP/PDF），对文件格式支持要求高
- `large_brand`: 采购流程中通常需要发送 RFQ 文件、验厂要求文档
- `manufacturer`: 可能需要分享原材料规格文件（多达数十页的技术标准文档）

### I-04: 响应时间预期的设定

```
[tags: persona:small_retail, persona:medium_brand, experience:novice, category:*, priority:high]
```

**规则目标:** 平台和供应商是否向买家设定明确的响应时间预期，避免买家因无响应而放弃或对平台失去信任。

**检查方法:**
1. 在询盘/聊天入口旁，检查是否显示供应商的「响应时间」（如「Usually responds within 24 hours」）
2. 检查响应时间数据的准确性说明（是历史平均值？还是承诺值？）
3. 提交询盘后，确认页面是否给出「预计 X 小时内收到回复」的预期
4. 检查当供应商在线时是否有明显标识（「Online Now」绿点）
5. 检查平台是否有「响应超时」的买家保护机制（如 Trade Assurance 中的响应时间要求）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 详情页显示供应商历史平均响应时间；询盘确认页有明确时间预期；在线状态有显示；超时有保护机制 |
| 可接受 | 部分页面显示响应时间；确认页有模糊时间说明（「工作日内」）；无在线状态但无大碍 |
| 有问题 | 响应时间数据不显示；询盘后无任何等待时间提示；买家不知道下一步该做什么 |
| 严重问题 | 平台显示的响应时间与实际严重不符（如显示「1小时」实际等待多天）；严重损害买家信任 |

**截图要求:**
- 必截：详情页中供应商响应时间信息的位置（及其可见性）
- 必截：询盘提交后的确认页面
- 如有问题：截图响应时间数据缺失或明显不准确的情况

**画像特别关注:**
- `small_retail` (novice): 首次询盘新手特别担心「石沉大海」，响应预期设定对其转化率影响最大
- `medium_brand`: 采购有时限要求，需要快速响应，检查是否可筛选「高响应率」供应商
- `large_brand`: 可能有专属 KAM（客户经理），检查是否有优先响应通道

### I-05: 区域特定合规信息的可见性

```
[tags: persona:medium_brand, persona:large_brand, category:*, industry:*, priority:medium]
```

**规则目标:** 针对买家所在地区的合规要求（GDPR、CE 认证、FDA 认证、环保标准等），平台或供应商是否在关键决策点提供足够的合规信息。

**检查方法:**
1. 以欧盟买家或美国买家的视角浏览商品，检查是否有地区相关的合规标签
2. 在详情页认证区域，检查 CE、FCC、FDA、REACH 等认证是否展示，且可以下载证书原件？
3. 检查数据隐私（GDPR）：平台在注册/登录流程中是否有数据处理说明？
4. 检查询盘表单中是否有「目标市场」字段，供应商是否可基于此提供合规建议？
5. 搜索「CE certification」关键词，检查平台是否有合规教育内容（Buy Guide / Compliance Guide）

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | 认证信息在详情页显眼展示，证书可下载；询盘表单有目标市场字段；平台有合规指南资源 |
| 可接受 | 认证徽章有展示但证书不可直接下载；无目标市场字段但买家可在询盘中说明 |
| 有问题 | 认证信息仅在商品描述文字中提及，无结构化展示；平台无合规教育资源 |
| 严重问题 | 认证信息虚假（展示未实际获得的认证）；或平台鼓励规避合规要求 |

**截图要求:**
- 按需截图：认证信息区域（有认证显示时截图）
- 如发现认证可下载：截图下载流程
- 如有合规问题：详细记录

**画像特别关注:**
- `large_brand`: 对产品合规有严格要求，检查是否有「Compliance Package」（证书套餐）一站式获取
- `medium_brand`: 进入新市场时需要合规引导，检查是否有针对性的 Buyer Guide
- `small_retail`: 通常不关注合规，但检查平台是否有最低限度的合规提醒（避免买家无意违规）

### I-06: Order Request 草稿功能的清晰度

```
[tags: persona:medium_brand, persona:large_brand, persona:service_provider, category:*, priority:medium]
```

**规则目标:** 平台的「Request for Quotation（RFQ）」或「Order Request」功能是否清晰易用，支持买家结构化地表达采购需求，匹配更多合适供应商。

**检查方法:**
1. 找到 RFQ / 「Post Buying Request」入口（通常在首页顶部或「Sourcing」区域）
2. 进入 RFQ 填写页面，截图完整表单
3. 检查表单的引导质量：字段是否有说明文字？是否有填写示例（Placeholder text）？
4. 检查提交 RFQ 后的流程说明：多少供应商会看到？多久会收到报价？如何选择报价？
5. 检查是否可以管理已发出的 RFQ（草稿保存/修改/撤回）
6. 检查 RFQ 的分发逻辑说明：是推给所有供应商还是精准匹配？

**评级标准:**
| 评级 | 标准 |
|------|------|
| 好 | RFQ 入口首屏可见；表单字段有详细说明和示例；提交后流程透明（时间/供应商数量预期）；支持草稿保存和修改 |
| 可接受 | RFQ 入口需寻找但可找到；表单基本清晰；提交后有确认但流程说明简略；无草稿但可重新发起 |
| 有问题 | RFQ 入口隐藏在二级页面；表单字段不清晰（不知道填什么）；提交后无流程说明 |
| 严重问题 | RFQ 功能无法使用（技术错误）；或 RFQ 发出后毫无响应且无状态追踪 |

**截图要求:**
- 必截：RFQ 入口位置（展示其在页面中的可见性）
- 必截：RFQ 填写表单（完整）
- 必截：提交后的确认和后续流程说明页面
- 如有问题：截图入口难以发现或表单不清晰的位置

**画像特别关注:**
- `service_provider`: 经常需要批量发 RFQ（为多个客户采购），检查是否支持「模板复用」或批量操作
- `large_brand`: 通常有详细的采购规格文档，检查 RFQ 是否支持附件上传和结构化规格填写
- `medium_brand`: 可能是第一次发 RFQ，检查新手引导是否足够（步骤说明、示例 RFQ 模板）
